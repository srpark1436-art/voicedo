# VoiceDo 음성 대화 가이드

## 아키텍처 구성

| 기능 | 엔진 | 비고 |
|------|------|------|
| 웨이크워드 감지 | Web Speech API | 항상 백그라운드 동작 |
| 음성 인식 (STT) | Gemini Live API | WebSocket 실시간 스트리밍 |
| 음성 응답 (TTS) | Web Speech API | 디바이스별 최적 한국어 음성 자동 선택 |

---

## 1. 웨이크워드

**지원 호출어:**
- "헬로 제니퍼" / "하이 제니퍼" / "안녕 제니퍼"
- "Hello Jennifer" / "Hi Jennifer"

**동작 조건:**
- 커맨드 모드 비활성 상태
- 음성 모달 닫힌 상태
- 로그인 완료 상태

---

## 2. 대화 프로세스 흐름

### 2-1. 기본 흐름

```
사용자: "헬로 제니퍼"
제니퍼: "무엇을 도와드릴까요?" → [음성 인식 대기]

사용자: "할일 추가할게"
제니퍼: "할일을 말씀해주세요" → [음성 모달 열림, 음성 인식 시작]

사용자: "내일까지 보고서 작성 높음 우선순위 저장해줘"
제니퍼: "저장했습니다" → [모달 닫힘]
```

### 2-2. 조회 흐름

```
사용자: "헬로 제니퍼"
제니퍼: "무엇을 도와드릴까요?"

사용자: "오늘 할일 알려줘"
제니퍼: "오늘 할일이 총 3건 있습니다. 읽어드릴까요?"

사용자: "응"
제니퍼: "1번, 보고서 작성, 마감일 4월 2일 수요일. 2번, ..." 

제니퍼: "확인할 내용이 더 있으신가요?"
사용자: "아니"
제니퍼: "알겠습니다" → [커맨드 모드 종료]
```

### 2-3. 상태 전이 다이어그램

```
[대기] ──웨이크워드──→ [커맨드 모드]
                          │
                  ┌───────┴───────┐
                  │               │
          [조회 명령]      [네비게이션 명령]
                  │               │
          "N건 있습니다.    필터/검색/삭제/
           읽어드릴까요?"   완료/정렬/추가
                  │               │
           Yes/No 대기     즉시 실행 후
                  │          종료 또는
                  │        "계속 확인?"
                  │               │
                  └───────┬───────┘
                          │
                  "더 있으신가요?"
                   Yes → [커맨드 모드]
                   No  → [대기]
```

## 3. 음성 명령어 목록

### 3-1. 커맨드 모드 (웨이크워드 후)

#### 조회 명령
| 음성 | 동작 |
|------|------|
| "오늘 할일 알려줘" | 오늘 마감 할일 조회 |
| "내일 할일 보여줘" | 내일 마감 할일 조회 |
| "이번 주 할일 알려줘" | 이번 주 할일 조회 |
| "다음 주 할일 알려줘" | 다음 주 할일 조회 |
| "3월 할일 알려줘" | 3월 전체 할일 조회 |
| "보고서 관련 할일 있어?" | 키워드 검색 조회 |
| "4월 15일 할일 알려줘" | 특정 날짜 조회 |

#### 네비게이션 명령
| 음성 | 동작 |
|------|------|
| "할일 추가" / "추가할게" | 음성 모달 열기 |
| "전체 보여줘" | 전체 필터 |
| "오늘 마감 보여줘" | 오늘 마감 필터 |
| "완료된 거 보여줘" | 완료 필터 |
| "보고서 검색해줘" | 키워드 검색 |
| "보고서 삭제해줘" | 키워드 매칭 할일 삭제 |
| "보고서 완료해줘" | 키워드 매칭 할일 완료 처리 |
| "우선순위 순 정렬" | 우선순위 높은 순 정렬 |
| "마감 순 정렬" | 마감일 빠른 순 정렬 |
| "최근 순 정렬" | 최근 등록 순 정렬 |
| "취소" | 커맨드 모드 종료 |

#### Yes/No 응답
| 의도 | 인식 키워드 |
|------|------------|
| Yes | 네, 예, 응, 맞아, 그래, 좋아, 읽어, 해, 당연, 물론 |
| No | 아니, 괜찮, 됐, 취소, 싫, 없, 끝, 필요없 |

### 3-2. 음성 모달 (할일 추가/수정)

#### 자동 감지
| 항목 | 음성 예시 | 감지 결과 |
|------|----------|----------|
| 우선순위 높음 | "긴급", "높음", "시급", "최우선" | priority: high |
| 우선순위 보통 | "보통", "일반" | priority: medium |
| 우선순위 낮음 | "낮음", "여유", "나중에" | priority: low |
| 오늘 마감 | "오늘" | deadline: 오늘 날짜 |
| 내일 마감 | "내일" | deadline: 내일 날짜 |
| 모레 마감 | "모레" | deadline: 모레 날짜 |
| N일 후 | "3일 후", "5일 뒤" | deadline: N일 후 날짜 |
| 요일 지정 | "금요일", "다음 주 화요일" | deadline: 해당 요일 |
| 날짜 지정 | "3월 15일", "4월 2일" | deadline: 해당 날짜 |

#### 음성 명령
| 음성 | 동작 |
|------|------|
| "저장해줘" / "저장" / "확인" | 할일 저장 |
| "삭제해줘" / "지워줘" | 할일 삭제 (수정 모드만) |
| "취소" / "닫기" / "그만" | 모달 닫기 |
| "다시" / "초기화" / "리셋" | 입력 내용 초기화 |

## 4. TTS 응답 목록

| 상황 | 응답 |
|------|------|
| 웨이크워드 감지 | "무엇을 도와드릴까요?" |
| 할일 추가 요청 | "할일을 말씀해주세요" |
| 할일 수정 모달 | "수정할 내용을 말씀해주세요" |
| 저장 완료 | "저장했습니다" |
| 수정 완료 | "수정했습니다" |
| 삭제 완료 | "삭제했습니다" / "할일이 삭제되었습니다" |
| 완료 처리 | "할일이 완료되었습니다" |
| 조회 결과 | "X 할일이 총 N건 있습니다. 읽어드릴까요?" |
| 조회 결과 없음 | "X 할일이 없습니다" |
| 계속 확인 | "확인할 내용이 더 있으신가요?" |
| 계속 Yes | "음성 명령을 계속해주세요" |
| 계속 No | "알겠습니다" |
| 인식 실패 | "다시 한번 말씀해 주시겠어요?" |
| 취소 | "취소했어요" |
| 필터 적용 | "X 할일이 N개 있습니다" |
| 정렬 적용 | "X순으로 할일을 정렬했습니다" |

## 5. 음성 예시 시나리오

### 시나리오 A: 빠른 할일 추가
```
"헬로 제니퍼"
→ "무엇을 도와드릴까요?"
"할일 추가할게"
→ "할일을 말씀해주세요"
"내일까지 팀 미팅 자료 준비 높음 우선순위 저장해줘"
→ [자동 감지: 마감일=내일, 우선순위=높음, 내용="팀 미팅 자료 준비"]
→ "저장했습니다"
```

### 시나리오 B: 할일 조회 및 완료
```
"헬로 제니퍼"
→ "무엇을 도와드릴까요?"
"오늘 할일 알려줘"
→ "오늘 할일이 총 2건 있습니다. 읽어드릴까요?"
"응"
→ "1번, 보고서 제출, 마감일 4월 2일 수요일. 2번, 회의 참석."
→ "확인할 내용이 더 있으신가요?"
"보고서 완료해줘"
→ "할일이 완료되었습니다"
→ "확인할 내용이 더 있으신가요?"
"아니"
→ "알겠습니다"
```

### 시나리오 C: 검색 및 삭제
```
"헬로 제니퍼"
→ "무엇을 도와드릴까요?"
"미팅 검색해줘"
→ "미팅 관련 할일이 3개 있습니다" → [검색 결과 표시]
```

## 6. 에러 처리

| 상황 | 동작 |
|------|------|
| 음성 인식 실패 | "다시 한번 말씀해 주시겠어요?" → 자동 재시도 (최대 2회) |
| 마이크 권한 거부 | "마이크 권한이 필요합니다" 에러 표시 |
| Gemini 연결 끊김 | "Gemini 연결이 끊어졌습니다" 에러 표시 |
| Yes/No 미인식 | 계속 대기 (타임아웃 후 재시도) |
